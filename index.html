시작 전 5가지 질문

⏱ 5분

Claude.ai를 처음 켜기 전 가장 많이 받는 5가지 질문에 먼저 답합니다.

Claude.ai와 Claude Code는 뭐가 다른가요? ▶
Claude.ai는 브라우저·앱에서 쓰는 채팅형 AI입니다. 글쓰기, 리서치, 파일 분석, 슬라이드·문서 생성, 대화형 도구 만들기에 강합니다. Claude Code는 터미널·IDE에서 쓰는 개발용 에이전트로, 소스코드를 직접 수정하고 명령어를 실행합니다. 이 가이드는 Claude.ai에 집중합니다.
ChatGPT랑 비교해서 뭐가 좋나요? ▶
긴 문서 분석(책 한 권 분량도 한 번에 처리 가능, Enterprise), 코드 품질, 한국어 자연스러움, Artifacts(코드·문서·앱을 실시간으로 만들어 보여줌), Projects(맥락 묶음 워크스페이스), Cowork(데스크톱 에이전트)가 강점입니다.
무료로 어디까지 쓸 수 있나요? ▶
Free 플랜으로 텍스트·이미지·코드 생성, 웹 검색, 파일 첨부, Artifacts, Memory(2026년 3월부터 전 사용자 무료)까지 모두 사용 가능합니다. 단 5시간 동안 사용할 수 있는 횟수 한도가 있고, 주로 Sonnet 4.6 모델로 연결됩니다. Opus 4.7과 Cowork는 Pro($20/월) 이상이 필요합니다.
내 대화가 모델 학습에 쓰이나요? ▶
기본적으로 대화는 학습에 사용되지 않습니다. 설정에서 명시적으로 동의해야만 학습에 활용됩니다. Team·Enterprise는 정책상 기본 비활성. 민감 정보는 그래도 마스킹·익명화하는 습관을 권장합니다.
한국어 잘 되나요? ▶
Opus 4.7·Sonnet 4.6 기준 한국어 글쓰기·요약·번역 모두 사실상 네이티브 수준입니다. 코드 주석·기술 문서·문학적 글쓰기까지 자연스럽고, 존대·반말·전문 톤도 지시대로 따릅니다.
01

가입 & 첫 로그인

⏱ 5분

웹에서 시작하기

  1. claude.ai 접속
  2. 이메일 또는 Google 계정으로 로그인
  3. 인증번호 입력 → 닉네임·직업 설정 → 첫 대화 시작

모바일 앱 (iOS / Android)

  • App Store / Play Store에서 "Claude" 검색
  • 웹과 동일한 계정으로 로그인 → 대화가 자동 동기화
  • 음성 모드·카메라 첨부·위젯 사용 가능

데스크톱 앱 (Mac / Windows) 권장

  • claude.ai/download에서 OS별 설치
  • 전역 단축키(Option+Space / Alt+Space)로 어디서든 호출
  • Cowork 기능은 데스크톱 앱 전용 (Section 15 참조)
💡

같은 계정이면 웹·모바일·데스크톱 대화 기록과 Projects가 모두 동기화됩니다. 대화는 어디서 시작해도 다른 기기에서 이어 쓸 수 있습니다.

04

첫 대화 — 좋은 프롬프트 작성법

⏱ 15분

같은 질문도 어떻게 묻느냐에 따라 결과가 천차만별입니다. 좋은 프롬프트의 4가지 요소를 봅니다.

1. 역할 (Role)

Claude가 어떤 입장에서 답해야 하는지 알려줍니다.

나쁜 예
이 글 어때?
좋은 예
너는 10년차 카피라이터야. 이 광고 헤드라인을 평가해줘.
브랜드 톤: 친근하지만 신뢰감 있는, 30대 워킹맘 타겟.

2. 맥락 (Context)

왜 이걸 하는지, 누가 볼 건지, 어떤 제약이 있는지.

예시
다음 주 임원 보고용 슬라이드 초안이야.
- 청중: 비기술직 임원 (AI 용어 모름)
- 목적: 도입 승인 받기
- 시간: 10분 발표, 슬라이드 7장
- 톤: 비즈니스 임팩트 중심, 기술 디테일 최소화

3. 구체적 작업 (Task)

"좋게 만들어줘"보다 "○○을 △△로 바꿔줘"가 훨씬 정확합니다.

4. 출력 형식 (Format)

  • 표·불릿·번호 매기기 중 어느 것
  • 분량 (단어 수, 문장 수, 슬라이드 장수)
  • 예시 1~2개 제공 ("이런 식으로 써줘"라고 보여주기)
💡

답이 마음에 안 들면 처음부터 다시 묻기보다 이어서 수정 요청하세요. "좀 더 짧게", "1번만 다시", "톤을 더 캐주얼하게" 같이. 모델이 이미 쌓은 컨텍스트를 활용합니다.

05

모델 선택 가이드 — Opus 4.7 / Sonnet 4.6 / Haiku 4.5

⏱ 10분

2026년 5월 기준 라인업입니다. 모델 선택은 우측 상단 드롭다운에서 언제든 바꿀 수 있습니다.

모델특기속도언제 쓰나
Opus 4.7최고 성능. 코딩·복잡 추론·디자인 감각보통중요한 글쓰기, 슬라이드 디자인, 복잡한 분석
Sonnet 4.6균형형. 일상 업무 90%빠름요약, 번역, 일반 글쓰기, 빠른 코딩
Haiku 4.5경량·초고속매우 빠름단순 질문, 분류, 빠른 응답이 필요한 일

실전 선택 기준

  • Opus 4.7: 보고서 초안, 슬라이드 자동 생성, 복잡한 코드 작업, 전략적 의사결정 보조
  • Sonnet 4.6: 이메일·블로그 작성, PDF 요약, 회의록 정리, 번역
  • Haiku 4.5: 사실 확인, 짧은 답변, 모바일에서 빠른 메모

Extended Thinking (확장 사고)

입력창 왼쪽 Search and tools 버튼 → Extended thinking 토글을 켜면 모델이 답하기 전 내부적으로 더 오래 추론합니다. 수학 문제, 코드 디버깅, 복잡한 의사결정에 효과적입니다.

  • 지원 모델: Claude 4 계열 (Opus 4.7, Sonnet 4.6)
  • 켜면 답변 위에 "Thinking" 인디케이터 + 타이머 표시
  • 펼치면 추론 과정 요약을 볼 수 있음
  • Opus 4.7은 Adaptive Thinking으로 자동 작동 (수동 토글 불필요한 경우도 있음)
💡

Extended thinking을 켜거나 끄면 새 대화로 분기됩니다. 한 대화 안에서 켰다 껐다 자유롭게 못 하니, 작업 시작 전에 결정하세요.

파일 첨부 활용

⏱ 10분

지원 파일 종류

유형예시활용
문서PDF, DOCX, TXT, MD요약, 비교, 질의응답, 번역
스프레드시트XLSX, CSV데이터 분석, 차트 생성, 인사이트 추출
이미지PNG, JPG, WebP, GIF설명, OCR, 디자인 피드백, 차트 해석
코드모든 텍스트 기반 코드 파일리뷰, 리팩터링, 버그 찾기
프레젠테이션PPTX슬라이드 재구성, 톤 변경

첨부 한도 (2026년 5월 기준)

  • 한 메시지당 최대 20개 파일
  • 파일당 최대 30MB
  • 이미지는 자동으로 압축 후 분석
  • Project에 업로드하면 모든 대화에서 재사용 가능

실전 팁

  • 긴 PDF는 첨부 후 "목차부터 보여줘"로 시작 → 관심 챕터만 깊게
  • 이미지를 첨부하면서 "한국어로 설명해줘"를 명시 (Claude는 첨부 언어 따라가는 경향)
  • 엑셀은 첫 행에 헤더 있는지 확인 후 첨부 → 분석 정확도가 다름
  • 스크린샷은 Cmd/Ctrl + V로 바로 붙여넣기 가능
⚠️

민감한 개인정보(주민번호, 카드번호, 비밀번호)는 첨부 전 반드시 마스킹하세요. 기본 정책상 학습에 안 쓰이지만 사고는 항상 사용자 측에서 시작됩니다.

07

Artifacts 마스터하기

⏱ 20분

Artifacts는 코드·문서·앱을 우측 패널에 실시간으로 렌더링해주는 기능입니다. 대화는 왼쪽, 결과물은 오른쪽에 살아 움직입니다.

무엇을 만들 수 있나

  • 웹 앱·인터랙티브 도구: 채팅창 안에서 직접 작동하는 웹 페이지나 도구
  • 문서 (.docx): 목차·표·스타일이 들어간 워드 파일 다운로드
  • 슬라이드 (.pptx): 차트·레이아웃 포함 파워포인트 다운로드
  • SVG·다이어그램: 로고, 플로우차트, 인포그래픽
  • 인터랙티브 차트 2026 신기능: 채팅 안에 인라인으로 차트 렌더링
  • 마크다운 문서: 블로그·뉴스레터 초안

예시 프롬프트

대화 예시
1) 첨부한 매출 엑셀 보고 월별 추이 차트를 Artifact로 만들어줘.
2) 우리 회사 OKR 1Q 회고 슬라이드 7장을 .pptx로 만들어줘.
   타이틀, 성과 3가지, 미달 2가지, 액션, 다음 분기 목표 순서로.
3) 직원 만족도 설문지를 인터랙티브 HTML로 만들어줘.
   5점 척도, 결과 즉시 차트로 보여주는 버전.

Artifact 다루기

  • Fork: 현재 Artifact를 복제해 다른 방향으로 발전
  • Rewind: 이전 버전으로 되돌리기 (실수했을 때)
  • Publish: 링크로 공유 (Pro 이상)
  • Download: 코드/파일을 로컬에 저장
  • Add to Project: 다른 대화에서 재사용 가능하게 묶기
💡

슬라이드를 만들 때 "디자인 톤: 미니멀·다크모드" 같이 디자인 가이드를 미리 주면 Opus 4.7이 디자인 감각을 살려 더 보기 좋은 결과를 만듭니다.

08

Projects — 컨텍스트 워크스페이스

⏱ 20분

Projects는 관련 파일과 지시사항을 묶어두는 영구 워크스페이스입니다. 그 안에서 시작한 모든 대화는 같은 자료·톤·규칙을 자동으로 따라갑니다.

만드는 법

  1. 좌측 사이드바 Projects → + Create Project
  2. 이름·설명·이모지 설정
  3. Project Knowledge에 PDF·문서·코드 업로드 (최대 ~200MB)
  4. Custom Instructions에 도메인·톤·규칙 작성
  5. 안에서 New chat → 모든 대화가 위 자료·규칙 자동 적용

Custom Instructions 예시

예시 — 마케팅 콘텐츠 프로젝트
역할: 너는 우리 회사 콘텐츠 마케터다.
브랜드 톤: 친근하지만 전문적, 이모지 최소, 반말 금지.
타겟: 30~40대 IT 종사자, 의사결정자.

규칙:
- 모든 글은 후크(1줄) → 본문 → CTA 구조
- 경쟁사 직접 비교는 피한다
- 출처 있는 통계만 인용
- 슬랭·유행어는 분기당 1회 이내

업로드된 PDF 3개는 우리 회사 톤·보이스·캠페인 사례다.
새로 만드는 글은 이 스타일을 반드시 따른다.

활용 예

  • 회사 위키 Project: 사내 정책 PDF + "신입에게 설명하듯이 답해줘"
  • 책 집필 Project: 기존 챕터 + 캐릭터 설정 + "1인칭, 현재시제"
  • 리서치 Project: 논문 50편 + "출처 페이지번호 포함해서 답해줘"
  • 강의·교육 Project: 강의 계획서 + 수강생 수준 + "쉽고 단계별로 설명"
💡

Project Knowledge는 검색 방식으로 동작해 모든 파일을 매번 다 읽지 않습니다. "○○ 문서 8페이지 표 기준으로"처럼 구체적으로 지목하면 정확도가 올라갑니다.

팀 공유 Team

Team·Enterprise 플랜은 Project를 팀원과 공유할 수 있습니다. 권한은 View / Comment / Edit 3단계. 신입 온보딩, 공통 SOP, 브랜드 가이드를 묶어두면 회사 차원의 AI 워크스페이스가 됩니다.

Memory — 대화를 가로지르는 기억 신기능

⏱ 15분

Memory는 Claude가 대화 사이를 가로질러 사용자에 대한 사실을 기억하는 기능입니다. 매번 "나는 사이드 프로젝트로 ○○ 하고 있어"를 다시 말할 필요가 없어집니다.

무엇을 기억하나

  • 직업·역할·전문 분야
  • 선호 톤·언어·답변 형식
  • 진행 중인 프로젝트·목표
  • 피드백 (예: "긴 설명 말고 결론부터")

저장 방법

두 가지 방식이 있습니다.

  1. 자동 추출: Claude가 대화 중 중요한 사실을 자동 저장 (저장될 때 화면 우측에 알림)
  2. 명시적 저장: "이거 기억해줘: 나는 한식 알레르기가 있어"

관리

  • Settings → Memory에서 저장된 사실 전체 목록 확인
  • 개별 삭제 / 수정 / 전체 초기화 가능
  • 특정 대화에서만 끄고 싶으면 입력창 메뉴 → "Don't use memory"

Projects vs Memory 차이

ProjectsMemory
범위해당 Project 안 모든 대화모든 대화 (Project 밖 포함)
저장 대상파일·문서·규칙사용자 자신에 대한 사실
예시회사 PDF, 코드베이스"나는 비건이다", "이메일은 한국어로"
⚠️

Memory에 민감 정보(주소, 전화번호, 가족 신상)는 저장하지 마세요. 저장하면 모든 대화에서 참조 가능해지므로, 공유 대화·screenshot·Publish 시 누출 위험이 커집니다.

Output Styles — 글쓰기 톤 저장 신기능

⏱ 10분

Output Styles(또는 Claude Design)는 자주 쓰는 출력 톤·형식을 저장해 한 번 클릭으로 적용하는 기능입니다.

기본 제공 Style

  • Normal: 기본값
  • Concise: 군더더기 없이 짧게
  • Explanatory: 단계별로 친절히 설명
  • Formal: 격식 있는 비즈니스 톤

커스텀 Style 만들기

  1. Settings → Styles → + Create Style
  2. 이름·설명·사용 시점 작성
  3. System prompt: 톤·금지어·형식 규칙 명시
  4. 샘플 답변 1~2개로 학습 (선택)
  5. 저장 → 입력창 옆 칩으로 언제든 적용

예시 — "임원 보고용" Style

지시사항 예시
모든 답변은 다음 구조로:
1) 한 줄 결론 (의사결정 포인트)
2) 근거 3가지 (각 1~2문장)
3) 리스크 또는 다음 행동

금지: 이모지, 인사말, "~인 것 같습니다" 같은 완충 표현.
숫자는 반드시 출처와 함께. 모르면 "확인 필요"라고 표시.
💡

Styles는 Projects보다 가벼운 단위입니다. "이메일용 / 슬랙용 / 블로그용 / 임원보고용"처럼 출력 형식 위주로 만들어두고, 입력창에서 골라 쓰는 게 가장 실용적입니다.

음성 모드는 2026년부터 모든 사용자에게 무료로 풀렸습니다. 모바일 앱·데스크톱 앱·웹 모두 지원.

음성 종류 (5종)

사용 시나리오

웹 검색 & 인용

⏱ 10분

Claude는 필요할 때 자동으로 웹 검색을 사용해 최신 정보를 가져옵니다. Free 포함 전 플랜에서 사용 가능.

자동 vs 수동

  • 자동: "오늘 환율", "최신 OpenAI 발표" 같이 시간 민감한 질문은 모델이 알아서 검색
  • 수동: 입력창 + 메뉴 → Web Search 토글로 강제 활성화
  • 비활성화: 토글 끄면 모델 내부 지식만 사용 (학습 컷오프 한계 있음)

인용 활용

검색 기반 답변에는 출처가 자동으로 붙습니다. 클릭하면 원문으로 이동.

활용 프롬프트
"2026년 상반기 한국 AI 스타트업 투자 동향을 조사해줘.
- 출처는 최근 3개월 이내 자료만
- 각 주장에 출처 링크 인라인으로
- 한국어 자료 우선, 영어는 보조"

리서치용 팁

  • "전문가 5명의 의견을 비교해줘" → 다양한 관점 검색
  • "2024년 자료는 제외" → 최신성 강제
  • "공식 발표·논문 우선, 블로그·트위터 제외" → 신뢰도 높임
⚠️

검색 결과도 환각이 0은 아닙니다. 의사결정에 쓸 정보는 원문 링크를 직접 클릭해 확인하세요. 특히 수치·날짜·인용문은 반드시 검증.

입력창 핵심 기능 — @ + Search and tools, Style 칩

⏱ 10분
⚠️

인터넷 글에 자주 보이는 /btw /rewind /fork /recap은 Claude Code CLI 전용 명령어입니다. Claude.ai 웹/앱 채팅에선 작동하지 않습니다. Claude.ai 입력창의 진짜 기능은 아래와 같습니다.

@ 멘션 — 파일 빠르게 지목

Project Knowledge에 올린 파일이 많을 때, 본문에 @를 입력하면 자동완성이 뜹니다. 원하는 파일을 골라 그 파일만 콕 집어 질문할 수 있습니다.

예시
@budget_2026.xlsx 의 Q1 합계만 뽑아줘.
@client_list.csv 와 비교해서 빠진 고객은 누구야?

+ 버튼 — 첨부 & Connectors

입력창 왼쪽 + 버튼을 누르면:

  • Upload file: PDF·이미지·CSV 등 첨부 (Section 06)
  • Add from Drive / Notion / GitHub 등: 연결된 Connectors에서 직접 가져오기 (Section 16)
  • Take screenshot (데스크톱 앱): 화면 캡처 즉시 첨부

Search and tools — 응답 방식 조절

입력창 왼쪽 Search and tools 메뉴 안 토글들:

  • Web search: 켜면 강제로 웹 검색 (Section 12)
  • Extended thinking: 답하기 전 더 오래 추론 (Section 05)
  • Drive / 기타 Connectors: 이 대화에서만 활성/비활성

Style 칩 — 톤 즉시 전환

입력창 옆 작은 칩으로 현재 적용된 Style이 표시됩니다. 클릭하면 다른 Style로 전환 가능 (Section 10). Style을 바꿔도 같은 대화가 이어지지만, 다음 답변부터 톤이 달라집니다.

드래그 & 드롭

  • 파일 드래그: 입력창에 끌어다 놓으면 첨부
  • Shift + 드래그: 이미 첨부된 항목에 추가
  • 이미지 붙여넣기: Cmd/Ctrl + V로 스크린샷 즉시 삽입
💡

모바일 앱은 입력창 아이콘이 약간 다릅니다. +는 사진/카메라/파일이 합쳐진 메뉴, 마이크 아이콘으로 음성 모드 바로 진입(Section 11). 데스크톱 앱은 추가로 전역 단축키(Option+Space / Alt+Space)로 어디서든 호출 가능.

트러블슈팅 & 자주 하는 실수

⏱ 15분
"한도에 도달했습니다" 메시지가 떠요. ▶
5시간 동안 사용 가능한 횟수 기준입니다. 우측 상단 시계 아이콘에서 리셋 시각을 확인하세요. 자주 막힌다면 Sonnet 4.6으로 모델을 내리고 (Section 05), 그래도 부족하면 Pro → Max 5× 순으로 업그레이드. 한도 소모량이 큰 Opus 4.7·긴 첨부·Cowork 작업을 줄이는 것도 효과적.
답변이 중간에 멈춰버려요. ▶
대부분 토큰 한도 초과입니다. "이어서 계속해줘"라고만 입력해도 모델이 끊긴 지점부터 다시 씁니다. 자주 발생한다면 출력 형식을 더 짧게 요청하거나, 한 번에 다 만들지 말고 단계로 쪼개세요.
PDF를 첨부했는데 "읽지 못한다"고 합니다. ▶
스캔본 PDF(이미지 형태)는 OCR이 안 될 수 있습니다. "이미지로 인식해서 OCR로 읽어줘"를 명시하거나, 무료 OCR 도구로 텍스트 추출 후 다시 첨부하세요. 또 한 가지 — 30MB 초과면 자동 차단되니 분할 필요.
같은 질문에 매번 다른 답을 줍니다. ▶
정상입니다. AI는 본질적으로 확률적입니다. 일관성이 중요하다면 Custom Instructions(Project) 또는 Style로 규칙을 못박으세요. "이 형식, 이 톤만 써"가 명시되면 변동성이 크게 줄어듭니다.
한국어 답인데 갑자기 영어 단어가 섞입니다. ▶
기술 용어·고유명사는 영어가 자연스럽기 때문입니다. 전부 한국어를 원하면 "모든 용어를 한국어로 번역, 영어 병기 금지"를 명시. 반대로 기술 문서라면 영어 그대로 두는 게 정확도가 높습니다.
생성된 .pptx / .docx 파일이 깨져요. ▶
한글 폰트 미설치 환경에서 자주 발생합니다. 다운로드 후 PowerPoint·Word에서 폰트 교체. 또는 프롬프트에 "폰트는 Malgun Gothic 사용"처럼 지정하세요.
파일 첨부가 한도를 너무 많이 먹습니다. ▶
같은 파일을 반복 첨부하는 게 가장 흔한 원인입니다. Project Knowledge에 한 번 올려두면 다음 대화부터 다시 첨부하지 않아도 됩니다 (Section 08).

Claude Cowork — 데스크톱 에이전트 2026 GA

⏱ 25분

Claude Cowork는 채팅창을 넘어 당신의 파일 시스템과 외부 서비스를 직접 다루는 데스크톱 에이전트입니다. Claude Code의 agentic 능력을 비개발 업무에 가져온 것. 2026년 1월 리서치 프리뷰, 4월 모든 유료 플랜에 GA.

채팅과 뭐가 다른가

ChatCowork
실행 위치브라우저·앱 안당신의 PC (파일 직접 접근)
지속성한 번에 한 답변여러 단계 작업을 자율 실행
도구웹 검색, 첨부파일시스템, 브라우저, 커넥터, 화면 제어
스케줄불가가능 (매일 아침 8시 등)

설치 & 시작

  1. 데스크톱 앱 (Mac/Windows) 설치 (Section 01)
  2. 앱 좌측 사이드바 Cowork 탭 선택
  3. 작업할 폴더 권한 부여 (예: ~/Documents/내회사)
  4. 커넥터 연결 (Notion, Slack, Gmail, Drive 등)
  5. 작업 지시 → 결과만 받기

실전 시나리오

예시 1 — 매일 아침 보고서
매일 아침 8시:
1. Gmail에서 어제 받은 이메일 중 답장 필요한 것 골라줘
2. Slack #general 채널의 어제 결정사항 정리
3. Notion의 "오늘 할 일" 페이지에 우선순위 매겨서 추가
4. 결과를 ~/Documents/daily/{날짜}.md 로 저장
5. 작업 끝나면 알림
예시 2 — 자동 갱신 대시보드
~/Documents/sales/sales.xlsx 와
HubSpot CRM, Stripe 결제 데이터를 연결해서
주간 매출 대시보드 HTML을 만들어줘.
다음 주부터 매주 월요일 아침 8시에 자동 갱신.

안전장치

  • 중요한 행동(파일 삭제, 메시지 전송, 결제) 전엔 항상 확인 요청
  • 접근 가능한 폴더는 처음에 명시한 것만
  • 활동 로그가 자동 기록 → 나중에 추적 가능
  • 언제든 Stop 버튼으로 중단
⚠️

Cowork는 실제 파일을 수정합니다. 처음 쓸 땐 반드시 테스트용 빈 폴더를 하나 만들어서 거기서 먼저 연습하세요. 중요한 폴더를 통째로 권한 주면 사고 발생 시 복구가 어렵습니다.

MCP 커넥터 — 외부 서비스 연결

⏱ 20분

MCP(Model Context Protocol)는 Claude가 외부 서비스에 직접 접근할 수 있게 해주는 개방형 표준입니다. 채팅·Cowork 모두에서 사용 가능.

공식 커넥터 (Settings → Connectors)

서비스가능한 작업
Gmail이메일 읽기·요약·초안 작성·답장 전송
Google Drive파일 검색·내용 읽기·새 문서 생성
Google Calendar일정 확인·생성·재조정
Notion페이지 검색·읽기·생성·업데이트
Slack채널·메시지 검색·DM 전송
Linear / Jira이슈 생성·상태 변경·검색
GitHub이슈·PR·코드 검색·리뷰 코멘트
Stripe / HubSpot매출·고객 데이터 조회

연결 방법

  1. Settings → Connectors → 원하는 서비스 옆 Connect
  2. 해당 서비스(Gmail, Notion 등) 로그인 화면이 뜨면 "허용" 클릭
  3. 입력창에서 @ 입력 → 커넥터 선택 → 사용

실전 예

대화 예시
@Gmail 어제 받은 메일 중 "계약" 관련된 것만 정리해줘.
@Notion 그 결과를 "거래 추적" 페이지 하단에 추가해줘.
@Slack #sales 채널에 요약을 공유 메시지로 보내줘 (전송 전 확인).

커스텀 MCP 서버 개발자

사내 도구나 자체 API를 연결하려면 MCP 서버를 직접 구현할 수 있습니다. modelcontextprotocol.io에 표준 스펙·SDK 공개. Python/TypeScript로 100줄 미만이면 기본 서버 구성 가능.

⚠️

커넥터 권한은 처음 인증할 때만 명확합니다. "메일 읽기"만 필요한데 "메일 전송"까지 허용하면 위험. 정기적으로 Settings에서 권한 점검하세요.

Computer Use & 원격 제어 프리뷰

⏱ 15분

Computer Use는 Claude가 화면을 보고 마우스·키보드를 조작해 일반 앱을 사용하는 기능입니다. API가 공식 커넥터를 지원하지 않는 서비스에 유용합니다. 2026년 3월 리서치 프리뷰로 공개.

⚠️

2026년 5월 현재 macOS 전용입니다. Windows·Linux 지원은 미정. Mac 사용자가 아니라면 이 섹션은 참고용으로만 보세요.

두 가지 모드

  • 로컬 Cowork 내부: 데스크톱 앱에서 Cowork가 직접 화면·앱 제어
  • 원격 제어: claude.ai/code 웹 또는 모바일 앱에서 PC에 있는 Claude Code 세션을 조종

원격 제어 시나리오

출근길 지하철에서 노트북에 두고 온 작업을 모바일로 이어가는 게 가능합니다. 데이터는 PC에 머무르고 채팅만 암호화된 채널로 전송됩니다.

  • 사무실 PC에서 Claude 데스크톱 앱을 켜두고 원격 연결을 활성화
  • 모바일 앱 → 해당 세션 선택 → 대화 이어가기
  • 파일·앱 설정·MCP 연결은 모두 PC에만 저장됨

안전 가이드

  • 로그인된 브라우저 세션은 모두 접근 가능 → 민감 탭은 사전에 로그아웃
  • 결제·이메일 전송 등 되돌릴 수 없는 행동은 항상 확인 단계 거치게 설정
  • 화면 녹화·캡처가 활동 로그에 남음 → 사후 검토 가능
🛑

Computer Use는 강력한 만큼 위험합니다. 회사 PC, 공유 PC에서는 활성화하지 마세요. 개인 작업용 머신에서만, 그것도 처음엔 비파괴적 작업(브라우징·읽기)만으로 신뢰를 쌓은 뒤 점진적으로 권한을 확장하세요.

팀 협업 — Team 플랜 활용 Team

⏱ 15분

5명 이상이 같이 쓴다면 Team 플랜이 비용·관리 모두 합리적입니다.

Team 플랜의 핵심 차별점

  • 공동 Projects: 회사 위키, 브랜드 가이드, 업무 매뉴얼을 한 곳에
  • 공동 Styles: 회사 톤 매뉴얼을 Style로 만들어 전사 공유
  • 중앙 청구: 개인 카드 정산 불필요
  • 관리자 대시보드: 사용량·활성 사용자·비용 추적
  • 데이터 정책: 학습 비활성화가 기본값

도입 단계 (권장)

  1. 파일럿 (1~2주): 3~5명 팀에 Team Standard 적용, 사용 사례 수집
  2. Projects 정리: 회사 문서·정책·브랜드 가이드를 Project Knowledge로 정리
  3. Style 표준화: 이메일·슬랙·블로그·보고서별 Style 만들고 전사 공유
  4. 온보딩 문서: "신규 입사자용 Claude 사용 가이드" 작성 (CL101 참조 가능)
  5. 월간 회고: 관리자 대시보드로 활용도 점검, 막힌 부서엔 케이스 스터디 공유

팀 운영 전 확인 사항

  • 민감 데이터 분류 가이드 (어떤 정보는 채팅에 못 넣는지)
  • 커넥터 권한 정책 (Gmail 전송 허용 여부 등)
  • Cowork 사용 범위 (개인 PC만? 회사 공유 폴더?)
  • 외부 공유(Publish) 승인 절차
💡

가장 자주 빠지는 함정: Team 플랜은 샀는데 Projects를 안 만들어서 결국 개인 채팅만 함. 도입 첫 주에 공동 Project 5개를 의무로 채워두면 활용도가 폭발합니다.

실전 — 나만의 Claude 워크플로우 구축

⏱ 40분

지금까지 배운 기능을 조합해, 직군별 "나만의 셋업"을 구축합니다. 본인 직군과 가장 가까운 케이스를 골라 그대로 따라 해보세요.

케이스 A — 콘텐츠 마케터

  1. Project: "브랜드 콘텐츠"
    • Knowledge: 브랜드 가이드, 톤 매뉴얼, 과거 베스트 콘텐츠 10개
    • Instructions: 타겟·금지어·구조 명시
  2. Styles 3개: "블로그용", "인스타용", "뉴스레터용"
  3. Memory: "나는 30대 워킹맘 타겟 브랜드 운영"
  4. Cowork 자동화: 매주 월요일 아침 → 지난주 콘텐츠 성과(Google Analytics, Instagram Insights) 가져와 회고 문서 작성
  5. 실행 루틴: 새 글 아이디어 → Project 안에서 대화 → Style 적용 → Artifact로 초안 → 수정 → Slack에 공유

케이스 B — 프로덕트 매니저

  1. Project: "제품 X 운영"
    • Knowledge: PRD, 로드맵, 경쟁사 분석, 최근 3개월 사용자 피드백
    • Instructions: "근거는 항상 첨부 자료에서, 추측 금지"
  2. Connectors: Linear (이슈), Slack (#product), Notion (회의록)
  3. Styles: "임원 보고용" (1줄 결론 + 근거 + 리스크)
  4. Cowork 자동화: 매일 아침 → 어제 들어온 이슈·피드백 우선순위 매겨 Notion에 추가
  5. 실행 루틴: 주간 우선순위 회의 전 "이번 주 논의된 내용 요약해줘"로 일주일 요약 → 미팅 중 음성 모드로 회의록 자동 정리

케이스 C — 1인 사업자·프리랜서

  1. Project: "내 사업"
    • Knowledge: 사업자등록증, 표준 계약서, 가격표, 포트폴리오
    • Instructions: "법률·세무 답변은 반드시 '전문가 확인 필요' 표시"
  2. Connectors: Gmail (견적·계약 메일), Calendar, Drive (포트폴리오)
  3. Styles: "고객 응대 메일", "견적서 초안", "SNS 공지"
  4. Cowork 자동화: 매일 아침 → 어제 받은 견적 문의 분류, 답장 초안 생성 (전송은 직접 확인 후)
  5. 모바일 활용: 외근 중 음성으로 견적 받아쓰기, 사진 첨부로 즉시 계약서 검토

공통 체크리스트 — 셋업 첫 1주일

  • [ ] 가장 자주 하는 업무 Top 3 정리
  • [ ] 각 업무마다 Project 하나씩 생성
  • [ ] Knowledge에 핵심 자료 5~10개 업로드
  • [ ] Custom Instructions에 톤·금지어·형식 작성
  • [ ] 자주 쓰는 Style 2~3개 등록
  • [ ] Memory에 "나는 누구, 무엇을, 어떻게 일하는지" 한 번 알려주기
  • [ ] Connectors 1~2개 연결 (Gmail부터 시작 추천)
  • [ ] Cowork 자동화는 2주차 이후 — 신뢰가 쌓인 뒤
✅

완벽한 셋업을 처음부터 만들려고 하지 마세요. 가장 자주 하는 업무 하나만 먼저 Claude로 옮기고, 매주 하나씩 추가하는 게 가장 빠른 길입니다.

Privacy & Incognito — 데이터 통제권

⏱ 10분

Claude.ai는 사용자에게 데이터 학습 허용 여부 선택권을 명시적으로 줍니다. 한 번 정해두면 끝이지만, 처음 한 번은 반드시 확인하세요.

학습 거부 (Opt-out) 설정

  1. 우상단 프로필 아이콘 → Settings
  2. Privacy 탭 (또는 claude.ai/settings/data-privacy-controls 직접 접속)
  3. "Help improve Claude" 토글을 끔
  4. 설정 즉시 적용. 기존에 학습용으로 들어간 데이터는 회수되지 않지만, 이후 대화는 학습에 안 쓰임
⚠️

2025년 9월 약관 개정으로 Free·Pro·Max 전 사용자가 한 번은 이 선택을 강제로 받게 됐습니다. 무심코 "Accept" 눌렀다면 켜져 있을 가능성이 높으니 지금 확인하세요. Team·Enterprise는 정책상 기본 비활성.

Incognito 모드 — 일회용 대화

중요한 주제(건강, 재무, 협상 전략)는 학습 토글과 무관하게 Incognito 모드로 시작하세요.

  • 새 대화 화면 우상단 👻 (유령 아이콘) 클릭 → Incognito 진입
  • 이 대화는 학습에서 항상 제외 (토글 켜져 있어도)
  • 대화 기록 보존 기간이 훨씬 짧음
  • Recents에 안 뜸 (직접 북마크 안 하면 다시 못 찾음)

예외 — 항상 검토되는 케이스

학습 거부했어도 안전 분류기가 위험 신호를 감지한 대화는 내부 신뢰·안전팀이 검토할 수 있습니다. 이는 정책 위반·악용 방지용으로, 정상 사용자에겐 사실상 무관합니다.

대화 삭제 & 데이터 다운로드

  • 개별 대화: Recents → 우클릭 → Delete
  • 전체 데이터: Settings → Account → Export data로 JSON 받기
  • 계정 삭제: 30일 유예 후 영구 삭제
🛑

주민번호·카드번호·비밀번호·의료 차트는 학습 거부 여부와 무관하게 애초에 입력하지 말 것. 정책상 안전해도 사고는 사용자 측 화면 캡처·공유에서 일어납니다.
